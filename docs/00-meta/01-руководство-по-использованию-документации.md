## Правила кодирования документов

Система организации документации является **иерархической**.

Каждый документ имеет строгое правило именования:

> `<код>-<название-документа>.md`

Где `<код>` — числовой идентификатор в справочной системе документации для эффективного поиска и группировки документов в единую иерархию.

**Принцип кодирования:**

-   Двузначный код = раздел (например, `00`, `10`, `20`).
-   Четырёхзначный код = подраздел (например, `00-01`, `00-02`).

**Пример для папки `00-meta`:**

-   `01-руководство-по-использованию-документации.md`
-   `02-правила-оформления-документов.md`

Названия пишутся на русском языке, так как основной контингент разработчиков и пользователей русскоязычный. Слова в названии разделяются дефисом (`-`).

## Правила оформления документов

У каждого документа есть **метаданные** (атрибуты), которые позволяют идентифицировать автора, дату создания, версию и ответственного.

Каждый документ должен иметь **оглавление** для удобной навигации.

### Обязательная структура документа

```markdown
# Название документа

> **Автор:** Имя Фамилия  
> **Ответственный:** Имя Фамилия  
> **Дата создания:** 21 апреля 2026 года  
> **Версия:** 1.0  
> **Статус:** Черновик / В работе / Готов

## Оглавление
1. [Первый раздел](#первый-раздел)
2. [Второй раздел](#второй-раздел)

## Первый раздел
Текст раздела...

## Второй раздел
Текст раздела...
```

## Версионирование документов

Документы могут изменяться со временем. Для отслеживания изменений используется **система управления версиями Git** (реализация — GitHub).

**Commit** — это зафиксированный «снимок» (snapshot) состояния файлов проекта в определенный момент времени. Он работает как контрольная точка или «сохранение» в игре, записывая изменения (добавление, удаление, редактирование файлов) из области подготовленных файлов (staging area) в историю репозитория.

### Правила оформления коммитов для документации

Все коммиты, касающиеся документации, должны начинаться с префикса `docs:`.

**Пример:**

```bash
git commit -m "docs: обновлён раздел 'Правила кодирования'"
```

**Форматы коммитов:**

| Тип изменения | Пример |
| :--- | :--- |
| Создание нового документа | `docs: создан 01-руководство-по-использованию` |
| Обновление существующего | `docs: обновлён раздел 'Версионирование'` |
| Исправление ошибок | `docs: исправлена опечатка в примере кода` |
| Удаление устаревшего | `docs: удалён устаревший шаблон документа` |

## Статусы документов

Каждый документ может находиться в одном из следующих состояний:

| Статус | Значение |
| :--- | :--- |
| **Черновик** | Документ создан, но ещё не завершён. |
| **В работе** | Документ активно пишется и дополняется. |
| **На ревью** | Документ ожидает проверки от других членов команды. |
| **Готов** | Документ завершён, актуален и готов к использованию. |
| **Архив** | Документ устарел, но сохранён для истории. |
