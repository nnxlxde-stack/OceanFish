## Общая архитектура

Система **OceanFish** построена по принципам **микросервисной архитектуры**.  
Это позволяет:

- Независимо разрабатывать, тестировать и деплоить подсистемы каждым участником команды.
- Легко масштабировать отдельные отделы предприятия.
- Обеспечить отказоустойчивость и независимость модулей.

## Основная программа

**Технология:** Kotlin Multiplatform (KMP)  
**Назначение:** центральная точка входа для пользователей и оркестратор всех микросервисов.

- Общая кодовая база (shared module) для бизнес-логики, моделей данных, валидации.
- UI — Kotlin Multiplatform (Compose Multiplatform для десктопа/веб/мобильного, если потребуется в будущем).
- Backend-часть главной программы — Ktor (JVM target).
- Выступает как **API Gateway** и единая точка аутентификации/авторизации.

## Микросервисы

Каждый отдел = отдельный микросервис:

| Отдел                          | Ответственный | Технология (рекомендуется)          |
|-------------------------------|---------------|-------------------------------------|
| Отдел кадрового учёта         | Портнягин Алексей    | Kotlin + Ktor / Spring Boot        |
| Отдел внешнеэкономической деятельности | Тожиев Роман | Kotlin + Ktor                       |
| Отдел мониторинга судов       | Нейкин Максим    | Kotlin + Ktor + WebSockets (реал-тайм) |
| Отдел промышленного рыболовства | Соколов Иван  | Kotlin + Ktor                       |
| **Интеграция + своя подсистема** | Тожиев Роман | **Kotlin Multiplatform + Ktor** |

Каждый микросервис:
- Открывает REST/gRPC API.
- Полностью контейнеризирован (Docker).

## Оркестрация и развёртывание

**Инструмент:** Docker Compose (docker-compose.yml в корне проекта)

- Один `docker-compose.yml` запускает все микросервисы + главную программу + базу данных + (при необходимости) Redis, Nginx и т.д.
- Локальная разработка: каждый разработчик поднимает только нужные ему сервисы.
- Продакшен: один compose-файл + volumes + networks.

## Взаимодействие между компонентами

- **Синхронное:** REST API + OpenAPI спецификации.
- **Асинхронное:** (будет определено позже) Kafka / RabbitMQ или простые HTTP callbacks.
- **Общие контракты:** OpenAPI / Protocol Buffers (gRPC) — будут вынесены в общий модуль KMP.
